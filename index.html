The app reads license data from localStorage keys set by the admin in Settings → License.
The PlantVisit app reads these localStorage keys. Share the license key with your client admin — they enter it in Settings → 🔑 License:
localStorage.setItem('pv_lic_key', 'PV-XXXX-XXXX-XXXX'); localStorage.setItem('pv_lic_org', 'Company Name'); localStorage.setItem('pv_lic_exp', 'YYYY-MM-DD'); // blank = no expiry localStorage.setItem('pv_lic_seats', '10'); // 999 = unlimited
ℹ The app's built-in Settings → License panel handles this automatically. Simply share the license key with the client's admin user.

📜 Required Firestore Security Rules

This panel signs in anonymously. If license creation fails with "permission-denied", your Firestore rules are blocking anonymous writes. Paste the rules below in Firebase Console → Firestore → Rules and click Publish.

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // ── License records — anonymous read + write (panel uses anon auth) match /pv_licenses/{licId} { allow read, write: if request.auth != null; } // ── Panel config (password hash, etc.) match /pv_panel_config/{docId} { allow read, write: if request.auth != null; } // ── PlantVisit app collections (adjust to your existing rules) match /{document=**} { allow read, write: if request.auth != null; } } }
⚠ The rule request.auth != null allows any authenticated user (including anonymous). For production, tighten this to specific UID(s) or use Firebase App Check.
